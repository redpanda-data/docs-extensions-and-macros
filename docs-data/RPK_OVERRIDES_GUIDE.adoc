= rpk Overrides Guide
:toc:
:toclevels: 3

This guide explains how to customize auto-generated rpk documentation using the `rpk-overrides.json` file.

== Overview

The rpk documentation is generated automatically from the `rpk --print-tree` command output. The `rpk-overrides.json` file allows writers to enhance, customize, and fix the auto-generated content without modifying the source code.

== File Location

The overrides file is located at:

----
docs-extensions-and-macros/docs-data/rpk-overrides.json
----

== Basic Structure

[source,json]
----
{
  "definitions": {
    // Reusable content blocks
  },
  "textTransformations": {
    // Global text transformations
  },
  "commands": {
    "rpk <command> <subcommand>": {
      // Command-specific overrides
    }
  }
}
----

== Command Overrides

Each command can be customized by adding an entry under the `commands` key using the full command path (e.g., `"rpk topic create"`).

=== Override Description

Replace the auto-generated description:

[source,json]
----
"rpk topic create": {
  "description": "Create one or more topics.\n\nThis command creates topics with the specified configuration."
}
----

=== Append to Description

Add content to the end of the auto-generated description:

[source,json]
----
"rpk topic create": {
  "appendToDescription": "For more details, see the topic configuration guide."
}
----

=== Override Flag Descriptions

Improve individual flag descriptions:

[source,json]
----
"rpk topic create": {
  "flags": {
    "partitions": {
      "description": "Number of partitions for the topic. More partitions allow higher throughput but use more resources."
    },
    "replication-factor": {
      "description": "Number of replicas for each partition. Higher values provide better fault tolerance.",
      "introducedInVersion": "v24.1.0"
    }
  }
}
----

Flag override properties:

* `description` - Replace the flag description
* `type` - Override the flag type
* `default` - Document the default value
* `deprecated` - Mark the flag as deprecated (boolean)
* `deprecatedMessage` - Explain the deprecation
* `introducedInVersion` - Document when the flag was added
* `cloudOnly` - Mark as only available in Redpanda Cloud
* `selfHostedOnly` - Mark as only available in self-hosted deployments

=== Exclude Flags

Remove flags from documentation:

[source,json]
----
"rpk topic create": {
  "excludeFlags": ["internal-flag", "debug-mode"]
}
----

=== Exclude Examples

Filter out specific examples from the source EXAMPLES section. Uses regex patterns that match against example descriptions or commands:

[source,json]
----
"rpk topic trim-prefix": {
  "excludeExamples": ["JSON file", "--from-file"]
}
----

This filters out any example whose description or command matches any of the patterns (case-insensitive).

== Content Array

The `content` array provides a flexible way to add various types of content at specific positions in the documentation.

=== Position Options

Content can be placed at these positions:

* `after_header` - After the page title
* `after_description` - After the description paragraph
* `after_usage` - After the Usage section
* `after_aliases` - After the Aliases section
* `after_flags` - After the Flags section
* `after_modifiers` - After format modifier sections
* `after_examples` - After the Examples section
* `before_see_also` - Before the See Also section
* `end` - At the end of the page

=== Include Partials

Include an AsciiDoc partial file:

[source,json]
----
"rpk topic delete": {
  "content": [
    {
      "type": "include",
      "position": "after_header",
      "path": "shared:partial$warning-delete-records.adoc"
    }
  ]
}
----

=== Add Notes, Warnings, Tips

Add admonitions:

[source,json]
----
"rpk cluster config set": {
  "content": [
    {
      "type": "note",
      "position": "after_description",
      "content": "Changes take effect immediately without requiring a restart."
    },
    {
      "type": "warning",
      "position": "after_flags",
      "content": "Modifying cluster configuration can affect all brokers."
    }
  ]
}
----

Supported admonition types: `note`, `warning`, `tip`, `caution`, `important`

=== Add Custom Sections

Add a custom section with AsciiDoc content:

[source,json]
----
"rpk cluster partitions balancer-status": {
  "content": [
    {
      "type": "section",
      "id": "balancer-status",
      "title": "Balancer Status",
      "position": "after_flags",
      "headingLevel": 2,
      "content": "[cols=\"1m,1a\"]\n|===\n|Value |Description\n\n|off |The balancer is disabled.\n|ready |The balancer is active but there is nothing to do.\n|===\n"
    }
  ]
}
----

Section properties:

* `id` - Unique identifier for the section
* `title` - Section heading text
* `position` - Where to place the section
* `headingLevel` - Heading level (2 = `==`, 3 = `===`)
* `content` - AsciiDoc content

=== Replace Source Sections

To replace content from a source section (like FIELDS, EXAMPLES, NOTES), use a section with the matching `id` and provide new `content`:

[source,json]
----
"rpk cluster partitions balancer-status": {
  "content": [
    {
      "type": "section",
      "id": "FIELDS",
      "content": "Custom replacement content for the FIELDS section."
    }
  ]
}
----

This replaces the auto-generated FIELDS section content with your custom content.

=== Exclude Source Sections

To completely remove a source section:

[source,json]
----
"rpk some-command": {
  "content": [
    {
      "type": "section",
      "id": "EXAMPLES",
      "exclude": true
    }
  ]
}
----

=== Add Structured Examples

Add examples with proper formatting:

[source,json]
----
"rpk topic create": {
  "content": [
    {
      "type": "examples",
      "position": "after_aliases",
      "items": [
        {
          "description": "Create a topic with 3 partitions",
          "code": "rpk topic create my-topic -p 3"
        },
        {
          "description": "Create a topic with custom configuration",
          "code": "rpk topic create my-topic -c retention.ms=86400000"
        }
      ]
    }
  ]
}
----

=== Add Examples with Output

Include expected output in examples:

[source,json]
----
"rpk cluster status": {
  "content": [
    {
      "type": "examples",
      "position": "after_aliases",
      "items": [
        {
          "description": "Check cluster status",
          "code": "rpk cluster status",
          "output": "CLUSTER\n=======\nredpanda.abc123\n\nBROKERS\n=======\nID    HOST        PORT\n0*    localhost   9092",
          "outputLanguage": "bash"
        }
      ]
    }
  ]
}
----

The `output` field creates a separate code block labeled "Output:" with the `.no-wrap` class for proper formatting.

=== Add Subsections

Nest content within sections:

[source,json]
----
"rpk cluster partitions move": {
  "content": [
    {
      "type": "section",
      "id": "partition-format",
      "title": "Partition Format",
      "position": "after_flags",
      "subsections": [
        {
          "title": "Basic Format",
          "content": "The basic format is `topic/partition:replica1,replica2,replica3`."
        },
        {
          "title": "With Core Assignment",
          "content": "Add core IDs using `broker-core` format: `topic/0:1-0,2-1,3-0`."
        }
      ]
    }
  ]
}
----

== Platform Restrictions

Mark commands as platform-specific:

[source,json]
----
"rpk redpanda tune": {
  "platforms": ["linux"]
}
----

Valid platforms: `linux`, `darwin`, `windows`

== Deprecation

Mark commands as deprecated:

[source,json]
----
"rpk old-command": {
  "deprecated": true,
  "deprecatedMessage": "Use 'rpk new-command' instead.",
  "deprecatedInVersion": "v24.1.0",
  "removedInVersion": "v25.1.0",
  "replacement": "xref:rpk-new-command.adoc[rpk new-command]"
}
----

== Version Information

Document when features were introduced:

[source,json]
----
"rpk connect run": {
  "introducedInVersion": "v23.1.0"
}
----

== See Also Links

Add cross-references to related pages:

[source,json]
----
"rpk topic create": {
  "seeAlso": [
    "xref:reference:rpk/rpk-topic/rpk-topic-delete.adoc[rpk topic delete]",
    "xref:manage:cluster-maintenance/manage-topics.adoc[Manage Topics]"
  ]
}
----

== Reusable Definitions

Define reusable content blocks:

[source,json]
----
{
  "definitions": {
    "tls-flags": {
      "tls-enabled": {
        "description": "Enable TLS for connections."
      },
      "tls-cert": {
        "description": "Path to the TLS certificate file."
      }
    }
  },
  "commands": {
    "rpk topic create": {
      "$refs": ["#/definitions/tls-flags"]
    }
  }
}
----

== Text Transformations

Define global text transformations:

[source,json]
----
{
  "textTransformations": {
    "inlineCode": [
      {
        "pattern": "(?<!`)\\b(STDOUT)\\b(?!`)",
        "replacement": "`stdout`",
        "description": "Convert STDOUT to inline code"
      }
    ],
    "replacements": [
      {
        "pattern": "redpanda-data",
        "replacement": "Redpanda",
        "description": "Normalize company name"
      }
    ]
  }
}
----

== Validation

Before committing changes, validate your overrides:

[source,bash]
----
# Run the generation with validation
npx doc-tools generate rpk-docs --output-dir ../docs/modules/reference/pages/rpk

# Check for validation errors in the output
----

The generator validates:

* JSON syntax
* Schema compliance
* Valid command paths
* Valid content positions
* Correct $ref references

== Examples

=== Complete Example

[source,json]
----
"rpk topic trim-prefix": {
  "excludeExamples": ["JSON file"],
  "content": [
    {
      "type": "include",
      "position": "after_header",
      "path": "shared:partial$warning-delete-records.adoc"
    },
    {
      "type": "note",
      "position": "after_description",
      "content": "This operation is irreversible. Trimmed records cannot be recovered."
    },
    {
      "type": "examples",
      "position": "after_aliases",
      "items": [
        {
          "description": "Trim records in topic 'foo' to offset 120 in partition 1",
          "code": "rpk topic trim-prefix foo --offset 120 --partitions 1"
        },
        {
          "description": "Trim records before a specific timestamp",
          "code": "rpk topic trim-prefix foo -o \"@1622505600\""
        }
      ]
    }
  ],
  "seeAlso": [
    "xref:reference:rpk/rpk-topic/rpk-topic-delete.adoc[rpk topic delete]"
  ]
}
----

== Best Practices

1. **Use structured examples** instead of blob content when possible
2. **Place includes at `after_header`** for warnings that affect the entire command
3. **Use `excludeExamples`** to filter unwanted examples rather than replacing the entire section
4. **Keep descriptions concise** - the auto-generated descriptions are often good starting points
5. **Test locally** before committing by running the generator
6. **Use definitions** for frequently repeated flag descriptions

== Troubleshooting

=== Content Not Appearing

* Check the position is valid for the content type
* Verify the command path matches exactly (including spaces)
* Ensure JSON syntax is valid

=== Section Not Being Replaced

* The section `id` must match the source section name (case-insensitive)
* The section must have `content` set to trigger replacement

=== Examples Not Being Filtered

* `excludeExamples` patterns are case-insensitive regex
* Patterns match against both description and command text
* Check regex syntax is valid
