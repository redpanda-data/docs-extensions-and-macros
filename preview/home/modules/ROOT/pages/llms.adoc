# Redpanda Documentation

> Redpanda is a streaming data platform for developers. Build real-time applications without the complexity of Apache Kafka.

## About This Documentation

Redpanda documentation is available at {site-url} and provides comprehensive guides for:

- **Self-Managed Redpanda**: Deploy and manage Redpanda on your own infrastructure (Kubernetes, Linux, Docker)
- **Redpanda Cloud**: Fully-managed serverless streaming platform
- **Redpanda Connect**: High-performance data pipeline tool (formerly Benthos)
- **Redpanda Console**: Web UI for managing and monitoring Redpanda clusters

## AI-Optimized Documentation Access

We provide multiple formats optimized for AI agent consumption:

### Interactive MCP Server

**https://docs.redpanda.com/mcp** - Model Context Protocol server for Claude Code

The Redpanda Documentation MCP server provides AI agents with tools to:

- Generate reference documentation (property docs, metrics, CLI commands, connectors)
- Access versioned content and API specifications
- Query documentation structure and navigate components
- Validate configurations and check compatibility

**Setup for Claude Code:**
```bash
npx doc-tools setup-mcp
```

After setup, restart Claude Code to load the server. You can then ask Claude to generate docs, check versions, or access any part of the documentation.

- **{site-url}/llms.txt**: This curated overview (llms.txt standard)

## Documentation Structure

### Components

**ROOT** (Self-Managed):: Installation, deployment, configuration, and operations for self-managed Redpanda
**cloud**:: Redpanda Cloud serverless platform documentation
**redpanda-connect**:: Data pipeline documentation (inputs, outputs, processors)
**redpanda-labs**:: Hands-on tutorials and example projects
**api**:: REST API reference (Admin API, HTTP Proxy, Schema Registry)
**home**:: Documentation home and cross-product resources

### Versioning

Self-managed documentation is versioned (24.3, 25.1, 25.2, etc.). Access specific versions:

- Latest: `{site-url}/current/page-path`
- Specific: `{site-url}/24.3/page-path`, `{site-url}/25.1/page-path`

Cloud and Connect documentation uses `current` for the latest version.

## Metadata

Each documentation page includes structured metadata:

- **Git dates**: `page-git-created-date`, `page-git-modified-date` (accurate commit history)
- **Version info**: Component version, Redpanda version, Console version, Operator version
- **Content type**: Tutorial, how-to, concept, reference (topic-type)
- **Learning objectives**: Measurable learning outcomes for tutorials
- **Personas**: Target audience (developers, operators, architects)
- **Categories**: Content categorization for filtering
- **FAQ structured data**: Schema.org FAQPage for search engines
